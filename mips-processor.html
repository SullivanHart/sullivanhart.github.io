MIPS Processor Project

My term project for computer architecture, where I designed and implemented a MIPS processor capable of running MIPS Assembly code, all created using VHDL.

The Project
In my Computer Architecture course, we were tasked with implementing a MIPS processor from scratch using VHDL. This project involved creating a processor capable of executing MIPS Assembly instructions.
The project was divided into several stages, starting with understanding the MIPS architecture, followed by designing and implementing the control and data paths, and finally integrating everything into a fully functional processor.
This project challenged my understanding of digital logic, VHDL coding, and the intricacies of processor design. It was an invaluable experience that deepened my knowledge of computer architecture.
Control Unit
The control unit is the heart of the processor, coordinating the operations of the datapath components. Designing the control unit was a complex task, requiring a thorough understanding of the MIPS instruction set and how each instruction affects the processor's state.
I implemented the control unit using finite state machines (FSMs) in VHDL. Each state corresponds to a phase of instruction execution, from instruction fetch to memory access and write-back.
By carefully managing these states, the control unit ensures that the processor executes instructions correctly and efficiently.
Data Path
The data path is a crucial component of the processor, responsible for data manipulation and execution of instructions. Designing the data path involved creating various registers, multiplexers, and arithmetic logic units (ALUs) to support different types of operations.
I implemented the data path in VHDL, ensuring it could handle various instructions and data formats as specified by the MIPS architecture.
The design of the data path was integral to the overall functionality of the processor, enabling it to perform complex operations and execute programs effectively.
